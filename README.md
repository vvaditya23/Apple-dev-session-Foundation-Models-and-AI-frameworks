# Foundation Models code-along workshop
Learn how to incorporate AI-powered features into your app using Foundation Models and other machine learning frameworks.

## Overview
> Note: This sample code project is associated with the Meet with Apple developer event
[Enhance your apps with the Foundation Models and machine learning frameworks](https://developer.apple.com/events/view/upcoming-events).

This sample project demonstrates how to build an intelligent meeting assistant app for iOS and macOS that helps teams capture, transcribe, and analyze meeting content.

The app combines multiple AI and ML frameworks to record and transcribe discussions, digitize whiteboard content, and provide intelligent insights from meeting materials.

During this code-along event, you'll use this sample to learn how to integrate Foundation Models, Speech, Vision, and Image Playground frameworks to build sophisticated AI-powered features that work together seamlessly.

## Features

The meeting assistant app demonstrates the following capabilities:

- **Audio transcription** – Convert spoken dialogue into text using the Speech framework and `SpeechTranscriber`.
- **Whiteboard digitization** – Scan and extract text from whiteboard photos using VisionKit for document capture and Vision for text recognition.
- **Visual content generation** – Generate visual aids and conceptual sketches with Image Playground and Foundation Models.
- **Intelligent summarization** – Condense lengthy meeting transcripts into actionable insights using Foundation Models with semantic chunking.
- **Document reasoning** – Extract key information and answer questions about meeting content using Foundation Models with Retrieval-Augmented Generation (RAG).
- **Structured output** – Ensure consistent, reliable responses from Foundation Models using guided generation techniques.
- **Context-aware tools** – Enhance Foundation Models through tool calling, to provide user-specific information.

## Getting Started

This project is structured as a hands-on learning exercise. You'll build the app incrementally, starting from a provided skeleton and implementing features step by step.

### Prerequisites

- Xcode 26.1 (latest beta) or later
- iOS 26.1 (latest beta) or later
- Mac with Apple Silicon (M1 or later)
- An iOS device for testing camera features
- Apple Intelligence active on both Mac and iOS devices
_Note:_ Apple Intelligence is not available on all iPhone models or in all languages or regions.

## Walkthrough

1. Meeting items can be recordings of team meetings, files, scanned documents or whiteboard photos.
2. Use the + button to create a new meeting item. Start with a recording.
   - The detail pane for a new recording is shown, press the Record button to start recording.
   - During recording, the transcribed text appears in the content view.
   - Press the button again to stop recording.
   - The finalized text will be shown and an image representing the topics of the discussion, generated by Image Playground, will appear at the top of the content.
3. Use the + button to add a file, or drag and drop a file to the list of meeting items.
   - Pick a text or pdf document, its contents will be added to the meeting material that can be used later for reasoning.
4. Use the + button to scan a document or whiteboard image.
   - Any text from the image will be recognized using the Vision framework, and added to the meeting material that can be used later for reasoning.
5. Once sufficient material has been collected, use the buttons in the toolbar to perform actions on the entire set of contents:
   - Extract action items from the recordings of team meetings.
   - Summarize all items into a concise recap.
   - Generate a timeline for the project being discussed, extracting key information such as dates, milestones, DRIs and progress for each item in the timeline.
   - Ask any question to the model, such as "When did we eventually decide to present our findings to the engineering team?".
